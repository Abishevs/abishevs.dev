# knowledge-system.md

The website is a curated public engineering knowledge system.

It is not a social media feed.

It is not a portfolio.

It is not a CV.

Instead, it documents the evolution of engineering knowledge over time.

Every piece of content contributes to a larger interconnected graph.

Examples include:

* Projects
* Project Journals
* Articles
* Work Experience
* Education
* Timeline Events
* Learning Notes
* Books
* Future content types

The same content may appear in multiple contexts without duplication.

For example:

Project
→ Project Index
→ Timeline Index
→ Work Experience
→ Journal
→ Related Articles

Timeline pages should visualize relationships rather than merely presenting chronological lists.

The Timeline Index is an alternative navigation system for the website.

All relationships are curated manually.

Nothing is inferred automatically unless explicitly configured.

Every published node is intentional.

The design should remain consistent with the existing engineering documentation philosophy.

The emphasis remains on exploration, context and discoverability rather than presentation.

# Anti Patterns

Never introduce new frontmatter fields
when equivalent existing fields already exist.

Prefer adapting templates to existing fields.

Example:

Use:

status
projectType

Do not replace them with:

state
kind
project_category

Never generate:

- Hero sections with large marketing slogans
- Call-to-action buttons
- Newsletter popups
- Pricing sections
- SaaS style cards
- Startup gradients
- Cyberpunk interfaces
- Excessive animations

Avoid:

- Large empty whitespace
- Oversized typography
- One-page landing page layouts

Prefer:

- Documentation layouts
- Notebook layouts
- Technical article layouts
- Information-rich designs
